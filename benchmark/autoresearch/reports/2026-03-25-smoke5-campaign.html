What E1 / E2 Actually Did

不是“变聪明”，而是“把实验搭对”

这两轮最重要的产出是研究基础设施，而不是 prompt 微调本身。

E1

E1-baseline-smoke5-k3-official 的任务，是第一次在 pre-warmed fast path 上拿到一条可信的 smoke-5 / k=3 基线。

  • 没有改 agent 行为
  • 验证了 run-experiment、结果统计、官方超时口径能一起工作
  • 给后续优化提供了第一条可比较的基准线

E2

E2-baseline-smoke5-k3-task-ledger 的任务，是把“一个总分”拆成“每个任务自己的稳定性画像”。

  • 新增 task-level ledger
  • 把 PPP / FFF / FFP / EPP 这种稳定性串持久化
  • 让每一轮 experiment 都能知道具体该打哪个任务
当前 campaign 真正的第一轮 agent 优化，是下一步的 E3。 当前候选 commit 为 759c251，核心假设是空 cwd 让 agent 在任务里失去路径上下文。
